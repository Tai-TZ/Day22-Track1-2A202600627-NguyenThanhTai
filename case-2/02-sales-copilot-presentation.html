01 · Bối cảnh

Trông mượt ≠ lookup đúng

Doanh nghiệp bán hàng online dùng Copilot đọc chat Zalo/FB → phát hiện SĐT/mã đơn → tra CRM/OMS → tóm tắt + gợi ý cho sales.

AI được phép

Phát hiện tín hiệu · Lookup nội bộ · Tóm tắt · Gợi ý next step · Nháp trả lời (sales duyệt)

AI bị cấm

Tự gửi tin · Tự chốt đơn · Sửa CRM · Auto-chốt 1 hồ sơ khi ambiguous · Bịa khách/đơn

02 · Unit of Work

04 · Workflow

Signal → lookup → warn → suggest

Khách nhắn (Zalo / FB / web chat)
        │
        ▼
Phát hiện tín hiệu (SĐT, email, mã đơn, mã KH)
        │
        ├─ Không đủ signal ──▶ insufficient_signal → hỏi thêm
        │
        ├─ CRM trả 2+ match ──▶ ambiguous → requires_human_pick
        │
        ├─ Owner đơn ≠ khách match ──▶ conflict_warning
        │
        └─ Single match + consistent ──▶ matched → summary + next step
                │
                ▼
        Nhân viên: dùng nháp / hỏi thêm / chuyển người
        (AI KHÔNG tự gửi)

08 · Code & LLM

13 code rules · 10 tiêu chí LLM judge

Code rules

Ambiguous → null IDs + warning · Not found → không bịa · Order owner ≠ customer → conflict · No auto_send fields · Signal trong transcript · requires_human_pick → disable nháp · Regression happy path

LLM judge (GPT-4o)

Summary accuracy · Groundedness · Ambiguity tone · Conflict surfacing · Order status in suggestions · Draft safety · Ask-for-more · Intent boundary · Over-confidence detection
